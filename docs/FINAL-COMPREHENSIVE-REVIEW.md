> 日期：2026-07-19

> 模式：Squad(×3视角) → 互评 → E7αβ对抗双审 → 调度中枢合成

## 一、Squad 三视角独立评审

### 👤 视角 A · 新生用户（第一次打开）

**我第一次打开这个网站：**

注册 → 看到一个很漂亮的 3 步引导（"你最近在学什么？""你的目标？""喜欢怎么学？"）→ 填完高数 → 首页出现了。

Agent 跟我说话了："欢迎来到三一！你提到了高数，我猜你可能需要高等数学、线性代数、概率论..."

**不错的地方：**
- Onboarding 设计精良，3 步引导不压迫
- Agent 的第一句话让我觉得"这个东西懂我"
- 视觉风格统一，字体舒服

**让我困惑的地方：**
- Agent 说完之后，我该怎么"添加科目"？页面上没有明显的"添加科目"按钮。Agent 说"告诉我科目名字"——但我不知道在哪里告诉它。左边的 AI 对话按钮太小了，我第一次没注意到。
- 首页那个"学习中心"标题下面直接是科目标签，但全是空的。我期待看到 Agent 推荐的那三个科目已经列在那里了。
- "一课一练"和"刷题"两个按钮并排，我不知道区别是什么。也没有 tooltip 解释。

**我最想要但没找到的：**
- Agent 推荐了科目，但我没办法一键添加。我需要去设置页手动创建——但 Agent 没告诉我怎么去。
- 我不知道"我现在该干什么"。Onboarding 之后应该有一个明确的下一步行动按钮，比如"开始第一节练习"。

### 👤 视角 B · 产品经理

**产品定位：7/10**

这个产品已经从一个"刷题工具"变成了"AI 学习伴侣"——这个转型是成功的。但我看到几个产品层面的断层：

**断层 1：Agent 的智能在"对话"里，不在"界面"里。** Agent 能诊断薄弱点、能生成计划、能分析错题——但这些能力都被藏在 AI 对话框里。用户需要主动打开对话框、主动打字、主动提问。首页、练习页、错题本——这些页面上看不到 Agent 的痕迹。Agent 的存在感在 UI 层太弱了。

**断层 2：Onboarding → 首次价值的路径太长。** 用户完成 Onboarding 后，需要：理解科目/章节的概念 → 找到一个章节 → 点"一课一练"→ 等 AI 生成讲义 → 等 AI 出题 → 开始做题。这条路径上每一步都可能流失。建议：Onboarding 完成后**直接在首页放 3 道 AI 生成的入门题**，用户不用理解任何概念就能先"玩一下"。

**断层 3：没有"社交证明"和"成就感"。** 用户做完一轮练习后，看到一个"正确率 67%"——然后就没了。没有"你今天比昨天进步了 15%"、没有"你已经超过 60% 的同专业同学"（匿名的）、没有连续打卡天数。学习本身是孤独的，产品需要给正反馈。

**值得肯定的：**
- Vault-as-Brain 架构是长期壁垒。用户在这里学得越久，Agent 越了解 ta，迁移成本越高。
- Skill 路由让自然语言交互成为可能——"帮我分析薄弱点"比任何菜单导航都直观。
- "顺便一提"的 Agent 行为设计是神来之笔——不是推送通知，而是自然嵌入对话。

### 👤 视角 C · 后端工程师

**架构评分：8/10**

这是我在本科项目里见过的最高水平的后端架构之一。Service 层职责分离清晰、Vault 文件系统设计优雅、Skill × Model 双路由解耦合理、Context Budget Manager 的设计预见了 token 经济问题。

**但有几个"只差临门一脚"的问题：**

**1. 错误处理仍然脆弱。** 虽然加了 `_tasks.json` 追踪，但用户端的错误提示仍然很粗糙。DeepSeek API 挂了 → 用户看到"AI 生成失败：502 Bad Gateway"——这是一行对用户毫无意义的错误消息。应该降级为"AI 暂时不可用，请稍后重试。你可以继续做已有题库的练习。"

**2. Chroma 数据库没有备份。** Vault 文件有 rsync 备份，但 Chroma 的数据（`data/chroma/`）没有被备份。如果磁盘损坏，所有 RAG 索引丢失——用户需要重新上传所有资料才能恢复搜索能力。修复只需在备份 cron 里加一行。

**3. 日志系统太简陋。** 所有日志都写到 `/tmp/quiz.log`——这是一个临时目录，系统重启后可能被清空。没有日志轮转、没有错误分级、没有结构化日志（JSON 格式）。debug 一个问题需要手动 grep 一个可能已经被清空的文件。

**4. 没有 health check 的深度检测。** `GET /health` 只返回 `{"status":"ok"}`。建议加上：数据库连接状态、Chroma 连接状态、DeepSeek API 可达性、磁盘空间、最近一次 cron 备份时间。

## 三、对抗双审

### E7-α · 独立审校（安全 + 逻辑 + 边界）

**安全审查：**

1. ✅ 路径遍历防护：`vault_manager.py` 的 `resolve()` + `startswith` 检查是正确的，能防止 `../../etc/passwd` 类攻击。
2. ✅ API Key 存储：用户 DeepSeek Key 存在 vault 文件里，不在 URL/日志中泄露。
3. ⚠️ 文件上传未做类型验证：`upload-material` 端点接受任何文件类型。虽然多模态 AI 会处理失败，但恶意用户可能上传 exe/zip 等垃圾文件。建议加白名单：仅允许 PDF/PNG/JPG/JPEG/GIF/WEBP。
4. ⚠️ 无 rate limit 在 Agent 端点上：`daily-plan` 和 `greeting` 是流式端点，每次调用消耗 DeepSeek token。如果没有 rate limit，恶意用户可以循环调用消耗 quota。建议加简单的 per-user rate limit（60秒内最多 1 次）。

**逻辑审查：**

5. ⚠️ `adaptive_difficulty.py` 的难度范围是 1-5，但 AI 出题 prompt 里没有把这个数字映射成具体指令。"difficulty=5"对 DeepSeek 来说意味着什么？建议在 `get_adaptive_prompt_hint` 里补充：difficulty 5 = "接近考研真题难度，需要综合运用多个知识点并进行创造性推理"。
6. ✅ reflect 的 AI 分析失败时正确回退到简单日志。`ai_analyzed` 字段现在准确反映状态。

**边界情况：**

7. ⚠️ 用户 vault 超过 1GB 时，`vault_manager.within_quota()` 返回 False，但没有任何端点检查这个返回值。用户仍然可以上传文件、写 journal——直到磁盘满了才报错。建议在 `write()` 和 `upload-material` 中主动检查 quota。
8. ⚠️ 如果用户的 DeepSeek API Key 余额用完了，所有 AI 功能静默失败。用户看到的是一系列不相关的错误消息。建议在 `get_vault` 时检查 API Key 有效性（一个轻量请求），并在首页展示"API Key 可能已失效"的警告。

### E7-β · 独立审校（UX 完整性 + 一致性 + 表达）

**UX 审查：**

1. 🔴 导航栏有 8 个入口（学习/好题/单词/笔记/复习/统计/设置/管理员），但"AI 助教"只有一个不显眼的对话框图标。对于"AI 学习伴侣"这个定位来说，AI 助教应该是**最显眼的入口**，而不是藏在角落。建议：把 AI 助教图标变大、加脉冲动画、或者在首页加一个醒目的"问 AI"输入框。

2. 🔴 练习完成页面没有任何"接下来做什么"的引导。用户做完题 → 看到正确率 → 页面停在那里。应该引导到：Agent 分析结果 / 错题复习 / 继续刷题 / 换个科目。

3. 🟡 全屏 AI 模式很好，但切换到全屏后，用户**回不去原来的页面**——只能缩小回侧边栏，然后手动关掉。应该在 ChatPanel 全屏模式加一个"返回"按钮，回到用户之前在看的内容。

4. 🟡 笔记模块在"学习"导航里看不到。笔记是独立导航项，但很多用户可能以为"笔记"在科目章节里。建议在章节页面加一个"记笔记"入口。

**一致性审查：**

5. 🟡 颜色语义不统一：正确 = 绿色、错误 = 红色、主观待判 = 黄色——这很好。但"掌握度星星"用的是自己的五级渐变色（红→橙→黄→绿→蓝），和其他地方的绿/红/黄不一致。建议掌握度统一用绿（高）/黄（中）/红（低）三级。

6. 🟡 加载状态不一致：每日计划有 spinner、AI 对话有"AI 正在思考..."、但 AI 出题（刷题）没有任何 loading 提示——用户点"刷题"后页面卡住 5-8 秒（等 AI 生成题目），然后突然跳转。建议加过渡动画。

## 七、结语

> 三一学习网站在 ~140h 的开发中，完成了从"AI 出题工具"到"AI 学习 Agent"的蜕变。Vault-as-Brain 架构、Skill 路由、RAG 检索、自适应难度、Agent 自学习——这五个子系统构成了一个完整的学习 Agent 底座。
>
> 当前 **7.3/10** 的评分意味着：**技术底座已经很扎实了，但用户第一次打开时感受不到 Agent 的存在。** 4 个 P0 全部指向同一个问题：Agent 在 UI 层太隐形了。
>
> 修完 P0（6h）后，新用户打开网站 → Agent 打招呼 → 推荐科目 → 一键添加 → 直接出题 → 做完题 → Agent 分析并引导下一步。这才是"AI 学习伴侣"应该有的体验。
>
> **预计修完后评分：8.0/10** ——一个可以作为本科毕设/竞赛项目/个人作品集核心的作品。
