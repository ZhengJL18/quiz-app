# 监控配置指南

## 方案一：Uptime Kuma（自部署，推荐）

`docker-compose.yml` 已包含。启动后访问 `http://你的服务器:3001`。

### 1. 添加监控

1. 打开 Uptime Kuma → 添加监控 → HTTP(s)
2. URL: `http://quiz-backend:8001/api/v1/health/detailed`
3. 检查间隔: 60 秒
4. 心跳重试: 3 次

### 2. 配置邮箱告警

1. 设置 → 通知 → 设置通知
2. 类型: 电子邮件 (SMTP)
3. 以 QQ 邮箱为例:
   - SMTP 主机: smtp.qq.com
   - SMTP 端口: 465
   - 安全: SSL/TLS
   - 用户名: 你的QQ邮箱
   - 密码: QQ邮箱的授权码（不是密码！QQ邮箱设置→账户→POP3/SMTP→生成授权码）
   - 发件人: 你的QQ邮箱
   - 收件人: 接收告警的邮箱

4. 回到监控编辑页 → 通知 → 选择刚配置的邮箱通知

### 3. 告警策略

- 连续 3 次失败 → 发邮件
- 恢复后 → 发邮件告知已恢复

## 方案二：UptimeRobot（免费，无需自部署）

如果不想自己跑 Uptime Kuma：

1. 注册 https://uptimerobot.com
2. 添加监控 → HTTP(s)
3. URL: `https://你的域名/api/v1/health/detailed`
4. 间隔: 5 分钟（免费版）
5. 告警联系人 → 添加邮箱

---

## 验证监控是否生效

```bash
# 正常状态
curl http://localhost:8001/api/v1/health/detailed
# {"status":"ok","checks":{"database":{"status":"ok","users":2,...}},"latency_ms":5.2}

# 模拟宕机：停掉后端
docker compose stop backend
# Uptime Kuma 在 60 秒后会检测到，连续 3 次失败后发邮件
```
